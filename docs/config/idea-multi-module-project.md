# IDEA 如何创建多模块项目

# 创建父项目

父工程主要作为一个容器，用来聚合所有的 `Demo`，本身不编写代码。创建步骤如下：

1. 打开 `IDEA`，点击 **New Project**。
2. 在左侧选择 **Maven**。
3. **Name**：输入项目名称，例如 `spring-learning-demos`。
4. **Location**：选择存放代码的硬盘位置。
5. **JDK**：选择常用的 `JDK` 版本（建议使用 `17`）。
6. **目录**：选择内部目录，这其实是 `Maven` 模板目录。选择后，下面的 **Archetype** 就可以选择对应模板。但现在一般不依赖模板，除非使用公司自定义模板。
7. **Archetype**：选择 `org.apache.maven.archetypes:maven-archetype-quickstart`，这是最简洁的模板，便于快速删除默认代码。
8. **组 ID**：例如 `com.公司名.项目名`。
9. **工件 ID**：与项目名一致。
10. **版本**：保留默认 `1.0-SNAPSHOT`，表示项目是新建并处于迭代阶段，一般无需修改。
11. 点击 **Create** 完成项目创建。

**重要操作**：创建好父工程后，你会看到根目录下有一个 `src` 文件夹。因为父工程只做管理用途，不需要写代码，请直接删除该 `src` 文件夹，这样可以避免误在父工程编写代码，保持项目结构清晰。

# 修改父工程的 pom.xml

打开根目录下的 `pom.xml` 文件，我们需要声明这是一个聚合工程，在 `<version>` 标签下面，添加一行打包方式的配置。

```xml
<packaging>pom</packaging>
```

这一步告诉 `Maven`，这个项目不是用来生成 `.jar` 或 `.war` 包的，而是用来管理其他项目的。

# 创建子模块 (Module)

现在我们可以开始创建具体的 `Demo` 了，比如 `demo-ioc`、`demo-aop`、`demo-web` 等。步骤如下：

1. 在项目左侧导航栏，右键点击父工程的名称。
2. 其他步骤和创建父工程类似，但需要**选择父模块**，子模块选择你自己定义的模块名称即可。
3. 点击 **Create** 完成创建。

创建完成后，`IDEA` 会自动检测并更新父工程的 `pom.xml`。如果你查看根目录下的 `pom.xml`，会发现多了一个 `<modules>` 标签，例如：

```xml
<modules>
    <module>your-demo</module>
</modules>
```

这样父工程就能统一管理所有的子 `Demo` 模块了。
